# DreamSpend

SwiftUI-приложение для игры "На что я потрачу..." с удваивающимся дневным бюджетом.

## Реализовано
- Экран "Сегодня": день, сумма, прогресс до максимума, переход к заполнению трат.
- Экран "Траты дня": добавление/удаление строк трат, итог/остаток, запрет сохранения при превышении лимита.
- Экран "История": список дней и детали по каждому дню.
- Экран "Настройки":
  - переключение языка (ru/en/de),
  - старт/максимум по текущему языку,
  - поведение на максимуме,
  - Approx FX (ручной курс),
  - локальные напоминания (включение + время).
- Экран "Ачивки": streak и игровые достижения.
- Полноэкранный "праздник" при достижении максимума + перезапуск.

## Данные и логика
- Модели: `DayEntry`, `SpendItem`, `Settings`, `Achievement`.
- Денежные значения хранятся в minor units (`Int64`) с банковским округлением.
- Календарная генерация дней по локальной дате.
- Streak reset при 2 пропусках подряд.
- При смене языка:
  - старые дни не меняются,
  - новые дни создаются в новой валюте,
  - фиксируется `conversionRateUsed` для первого дня после переключения.

## Проверка
- `swiftc -typecheck` для App/Core/Features проходит.
- `XCTest`-файлы добавлены в `DreamSpend/Tests/Unit/*`.

## Важно
- В репозитории пока нет `xcodeproj`, поэтому для полноценного `xcodebuild test` нужно создать проект и подключить эти файлы к app/test target.

## Запуск на iPhone 17 из CLI

Чтобы избежать ошибки `CoreSimulator runtime` при запуске из терминала, используйте запуск по `destination id` (а не по фиксированному `OS=...`).
Команды ниже выполняйте из корня репозитория.

- Сборка под симулятор `iPhone 17`:
  - `./scripts/ios_sim_runner.sh build`
- Тесты на симуляторе `iPhone 17`:
  - `./scripts/ios_sim_runner.sh test`

Скрипт сам находит актуальный `iPhone 17` в `xcodebuild -showdestinations` и использует `-destination id=...`.

Если нужно другое устройство, передайте переменную `DEVICE_NAME`, например:
- `DEVICE_NAME="iPhone 17 Pro" ./scripts/ios_sim_runner.sh test`
